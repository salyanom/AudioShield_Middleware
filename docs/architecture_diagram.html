System Architecture v1.1.0

🛡️ AudioShield

A model-agnostic, five-stage hybrid security middleware for real-time Voice-AI pipelines. Intercepts adversarial prompt injections, acoustic decoder obfuscation, and unsafe LLM outputs before delivery.

S1
Stage 1 — Input Policy Check
DistilBERT · distilbert-base-uncased (fine-tuned)

Classifies the raw user transcript T for unsafe intent before the LLM processes it. If the input unsafe probability Pinput exceeds the threshold (0.80), the pipeline short-circuits immediately — the LLM never sees the dangerous prompt. This provides ~320ms BLOCK latency for clearly malicious injections.

policy_checker.py Pinput ≥ 0.80 → BLOCK 253-example fine-tuned
If Safe → Forward T
S2
Stage 2 — LLM Response Generation
Ollama (Llama 3.1 · Phi-3) · OpenAI · vLLM · Stub

Forwards the safe transcript T to the configured LLM backend for response generation. AudioShield is fully model-agnostic — it adapts to local Ollama servers (Llama 3.1:8b, Phi-3), hosted OpenAI endpoints (GPT-4o), vLLM/LM Studio gateways, or the deterministic Stub provider for reproducible offline evaluation.

llm_engine.py Model-Agnostic Configurable via ENV
Raw Response R
S3
Stage 3 — Output Policy Check
DistilBERT · distilbert-base-uncased (fine-tuned)

Re-evaluates the generated LLM response R through the same fine-tuned DistilBERT safety classifier. This catches cases where a benign-looking input tricks the LLM into generating dangerous output (e.g., jailbreaks, credential leakage). Outputs the output unsafe probability Poutput used in the hybrid risk formula.

policy_checker.py Poutput → Risk Formula
T + R + Waveform A
S4
Stage 4 — Dual Context Verification
MiniLM (all-MiniLM-L6-v2) + CLAP (laion/clap-htsat-unfused)

Performs two independent semantic alignment checks:
① Text-to-Text (simt): Cosine similarity between transcript T and response R via SentenceTransformer MiniLM. Detects prompt injections that cause the LLM to answer an unrelated hidden question.
② Audio-to-Text (sima): Cross-modal cosine similarity between the raw audio waveform A and response R via LAION CLAP. Detects acoustic-level obfuscation (pitch shifting, noise injection, echo) invisible at the transcript level.

context_verifier.py audio_embedder.py sim_t → MiniLM sim_a → CLAP 512-dim
Poutput , simt , sima
S5
Stage 5 — Hybrid Decision Engine
Risk Score ℛ = wp·P + wc·(1−simt) + wa·(1−sima)

Computes a unified risk score ℛ ∈ [0, 1] by linearly combining the DistilBERT unsafe probability, text semantic deviation, and audio-text cross-modal deviation using configurable importance weights. The score is evaluated against two thresholds to produce one of three decisions:

middleware.py wp=0.40, wc=0.35, wa=0.25 τmit=0.40, τblk=0.60
✅
ALLOW
ℛ < 0.40
Deliver raw response R directly to the user. The input and output are verified safe with high confidence.
🛡️
MITIGATE
0.40 ≤ ℛ < 0.60
Apply Smart Mitigation (LLM rewrite) + Rule-Based Sanitizer (PII, secrets, commands, URLs, code blocks, file paths) before delivery.
🚫
BLOCK
ℛ ≥ 0.60
Return a static security fallback message. The raw response is never delivered. Logged to JSONL audit trail for forensic review.

Mathematical Foundation

Hybrid Risk Score Formula

ℛ = wp · Punsafe + wc · (1 − simt) + wa · (1 − sima)
Punsafe
Output Unsafe Probability DistilBERT classifier confidence that the LLM response is unsafe. Range [0, 1].
simt
Text-to-Text Similarity MiniLM cosine similarity between transcript T and response R. Range [−1, 1].
sima
Audio-to-Text Similarity CLAP cross-modal cosine similarity between raw waveform A and response R. Range [−1, 1].
wp,c,a
Importance Weights Normalized weights summing to 1.0. Default: wp=0.40, wc=0.35, wa=0.25. When CLAP is unavailable, wp and wc are renormalized.
